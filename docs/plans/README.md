| [2026-06-18-session-command-ergonomics](2026-06-18-session-command-ergonomics-design.md) | shipped (v0.34.0) | **Full** consolidation of the worktree work-unit on **`session`**: vocabulary, code, and on-disk layout. Adds a primary `my session start\|join\|resume\|list\|status\|finish` surface (with the new collaboration verb `my session join <id> <harness>`); keeps `my ai --new-session/--session/-r` as bot-stable launch shortcuts; demotes `my work ...` to a deprecated alias slated for removal. Renames the layout (`work/<id>` -> `sessions/<id>`, `my/work/<id>` -> `my/session/<id>`, noun-free ids) and adds a lazy, idempotent, safe **migration** for existing sessions (same-fs `git worktree move` preserves dirty work; orphans go to `my doctor`) so the `work` noun is freed everywhere. Core UX fix: creating a session surfaces id/path + "join another harness" + finish hints (stderr before exec; `--json` carries `launch_command`/`join_command`/`finish_command`). Human-short verbs, bot-wide JSON. |
